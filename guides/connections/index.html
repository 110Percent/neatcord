Custom Profile Connections with PUT Requests

Learn how to add custom text to your Discord profile with custom social media accounts.

When it comes to profiles, Discord is a bit lackluster when you look at customization. The only real things that you can modify completely are your username and your avatar. You can add notes to profiles, but those only apply to your own client and not the profile itself. Eventually, all you have left are the social media connections at the bottom of your profile, and those are controlled by the services you use... or are they?

Using a specific HTTP method called a PUT Request, we can trick Discord into thinking we have different game/social media (Skype, Battle.net, League of Legends, Steam, etc.) accounts than we actually do, and therefore we can trick our profiles into displaying custom text for other users to see.

If you already know HTTP methods work, skip to the next section to get started. Otherwise, please continue normally. It's better to understand fully than be confused later.

What are HTTP methods?

HTTP methods are what web browsers and web services often use to communicate with each other when they want to send single requests. When you load a webpage, your browser makes a request to the server it's trying to access, then the server sends the data in its response. When you send a message on Discord or submit a post to Reddit, a different kind of method is used. When you're on the web, HTTP requests are everywhere.

Why am I using this sort of request?

In this guide, we'll be sending a specific type of HTTP request to Discord's servers. When Discord receives the request, the server interprets the request as a new game/social media account being added to your profile, and adds it with the parameters you provide.

Requirements

To complete this guide, you will need the following:

  • Any Discord client (Desktop or Browser)
  • Postman API Request Builder
Setting up Postman and the request
  1. Open Postman using the method for your operating system (or Chrome, if you installed the Chrome app).
  2. Select the dropdown menu under the tab indicator, and set the value to PUT.
  3. In the Headers section, add the following headers:
    4. Key Value
    Cache-Control no-cache
    Content-type application/json
  4. Select the Body section next to the Headers label.
  5. Check the raw radio button.
  6. Set the dropdown menu to the right of the radio buttons to JSON (application/json)
  7. In the code box, paste the following:
    8. {
	"name": "CUSTOMTEXTHERE"
}
    • Replace CUSTOMTEXTHERE with the text you want to show on your profile.
  8. In the URL bar above the Headers and Body labels, paste the following URL:
    9. https://canary.discordapp.com/api/v6/users/@me/connections/PLATFORM/NAME
    • Replace PLATFORM with one of the following:
      • leagueoflegends
      • steam
      • skype
      • battlenet
    • Replace NAME with any alphanumeric string (letters and numbers).
      • Remember to change this value every time you add a new connection.

Do NOT click the Send buton. Proceed to the next step.

Fetching your token

In order to link the request to your account, you need to get your Discord token. Your token acts as a sign-in key for the server to identify and verify your account.

Do NOT share your token with anyone. Your token can be used to hijack your account should other people find it. Always keep it private.

  1. Open the Discord Client.
  2. Press Ctrl+Shift+I (Cmd+Shift+I on Mac) to open the Developer Tools.
  3. At the top of the dialog, open the Application panel.
  4. On the left, click on the dropdown arrow next to Local Storage, then click on https://discordapp.com.
  5. You should see a table on the dialog. At the bottom of the table, select the value across from token.
  6. Copy the value from Step 5 to the clipboard.
  7. Close the Developer Tools dialog with the x at the top-right.
Entering your token into the request
  1. In Postman, open the Headers panel.
  2. Add the following header to your request:
    3. Key Value
    Authorization TOKENHERE

    Replace TOKENHERE with the token you copied. Remember to remove the quotation marks if you copied them as well.

Sending the request

Once you've verified that the name property in the request body is set to the custom text you want, click the blue Send button in the top-right of the page.

Check the Response panel. The server should return with a response code of 204. This means the server has successfully processed the request, but has not returned anything. Don't worry, this is normal.

Showing the connection on your profile

The server's returned with a proper code, meaning the connection has been added to your account. Great!

  1. Open the Discord client and open the user settings by clicking on the gear next to your profile in the bottom left.
  2. Navigate to the Connections section.
  3. You should see your new connection in the list with the icon matching the service you selected.
  4. If there is one switch on the connection, click it to show it on your profile. If there are two switches, click the leftmost switch.
Managing connections

You now have a new connection on your profile! Open your profile by navigating to a channel/DM, clicking on your avatar/username and showing your profile.

If you want to add multiple connections, be sure to change the NAME parameter of the URL in the request. If you can't remember what you had previously set it to, check the previously sent requests on the left of Postman and look at the URL in the top section of the panel.

If you want to hide a connection, go back to the Connections section of your settings and hide the connection using the switch.

To delete a connection click the X on the top-right of the connection. It should disappear from the list, and you'll need to send a new request to get it back.
